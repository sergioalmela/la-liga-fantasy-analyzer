Host upstream:

```text
https://fantasy-api.llt-services.com/api
```

Prefijo de Primera División:

```text
/v1/competition/1
```

## Lecturas permitidas por el proxy

```text
/v4/user/me
/v3/teams-master
/v1/competition/1/leagues
/v1/competition/1/leagues/{leagueId}/standing
/v1/competition/1/leagues/{leagueId}/standing/{week}
/v1/competition/1/leagues/{leagueId}/activity/{page}
/v1/competition/1/leagues/{leagueId}/teams/{teamId}
/v1/competition/1/teams/{teamId}/money

/v1/competition/1/teams/{teamId}/lineup/week/{week}
/v1/competition/1/league/{leagueId}/market
/v1/competition/1/players
/v1/competition/1/player/{playerId}/market-value

/v1/competition/1/week/current
/v1/competition/1/calendar?weekNumber={week}
/stats/v1/competition/1/stats/week/{week}
```

Solo se permiten los parámetros `x-lang=es` y `weekNumber` numérico. El proxy
rechaza URLs externas, traversal y rutas legacy.

## Publicación de la plantilla

El proxy permite únicamente las dos escrituras necesarias para añadir o renovar
jugadores en el mercado:

```text
POST   /v1/competition/1/league/{leagueId}/market/sell
DELETE /v1/competition/1/league/{leagueId}/market/{marketId}/delete
```

El `POST` acepta exclusivamente `playerId` y un `salePrice` entero positivo. Las
mutaciones exigen sesión, origen same-origin y confirmación previa en la
interfaz. El flujo se ejecuta de forma secuencial y muestra fallos individuales.

## Normalización

Los adaptadores convierten IDs y cantidades numéricas recibidas como strings,
desenvuelven arrays cuando la API usa `elements` o `leagues`, normalizan el
standing y enriquecen `playerMaster.teamId`, `localId` y `visitorId` con
`/v3/teams-master`.

Estas lecturas alimentan el panel de jornada, que combina la alineación de la
semana seleccionada, el calendario, las estadísticas disponibles y el standing
semanal. La evolución suma los puntos por jornada obtenidos de
`standing/{week}`; si la API cambia `points` a un total acumulado, habrá que
ajustar ese cálculo. Un `404` de estadísticas antes de que empiece la jornada se
interpreta como sección todavía no disponible.

`/v4/user/me` se usa únicamente para validar la sesión upstream y mostrar el
nombre del mánager. La ficha de jugador usa
`/player/{playerId}/league/{leagueId}` y muestra solo los campos que pasan la
normalización runtime.

El histórico oficial de valor usa
`/v1/competition/1/player/{playerId}/market-value`. Mercado y Oportunidades
calculan con él las variaciones de 1, 3 y 7 días y un impulso ponderado
(`40 %`, `40 %`, `20 %`). Las peticiones se limitan a seis concurrentes y se
reutilizan durante seis horas para no sobrecargar la API. Mis jugadores usa el
mismo análisis para señalar tendencias bajistas, sin ejecutar ventas
automáticas.

Un body con forma inesperada se devuelve como error de contrato y nunca llega a
un `.map()` de React. Los status upstream, incluidos `401`, `404`, `500` y
respuestas vacías `204`, se preservan.

## Funciones desactivadas

- Pujas, ofertas, blindajes, clausulazos y cambios de alineación.
- Endpoints legacy de histórico como `/v3/player/{playerId}/market-value`, que
  ya no representa el contrato de la competición actual.

Antes de habilitar otra mutación se necesita validar el contrato actual, limitar
el endpoint y body en el proxy, añadir protección CSRF, mostrar una confirmación
específica con el importe y probar únicamente con autorización explícita.
